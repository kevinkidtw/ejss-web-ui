## 環境需求與啟動

### 需求

- Node.js 18 以上
- 現代瀏覽器（Chrome / Edge / Firefox / Safari）

### 安裝與啟動

```bash
# 安裝相依套件
npm install

# 啟動開發伺服器
npm run dev
```

啟動後開啟瀏覽器前往 `http://localhost:5173`（或終端機顯示的對應埠號）。

### 建置正式版

```bash
npm run build
```

產出的靜態檔案位於 `dist/` 資料夾，可直接部署至任何靜態網頁主機。

## 快速上手：建立第一個模擬

以**簡諧運動**為例，帶領完整流程。

### 步驟一：設定模型變數

在左側積木面板點擊「**+ 新增模型變數**」，新增以下四個變數：

| 名稱 | 型別 | 初始值 | 說明 |
|------|------|--------|------|
| `x` | 實數 | `2` | 質點位置（公尺） |
| `vx` | 實數 | `0` | 質點速度（公尺/秒） |
| `k` | 實數 | `8` | 彈簧係數（N/m） |
| `m` | 實數 | `1` | 質量（kg） |

### 步驟二：寫微分方程

點擊「**+ 新增方程組**」，填入以下兩條方程：

```
d[x]/dt  = vx
d[vx]/dt = -k * x / m
```

- 時間步長 dt 建議填 `0.02`
- 積分方法選「RK4 四階法（推薦）」

### 步驟三：新增視覺元件

在右側「元件列表」點擊「**新增**」，依序加入：

1. **DrawingPanel（繪圖面板）**
   - X 軸最小值：`-4`，最大值：`4`
   - Y 軸最小值：`-3`，最大值：`3`

2. **Shape2D（質點/形狀）**
   - X 位置：`x`（填入變數名稱，讓它動起來）
   - Y 位置：`0`
   - X 半徑 / Y 半徑：`0.3`

3. **Trail2D（軌跡線）**（選填）
   - X 位置：`x`
   - Y 位置：`0`

### 步驟四：執行模擬

按右側舞台上方的 **▶ 執行** 按鈕，即可看到質點來回振盪。

> 也可點擊工具列的「🎮 模擬舞台」全螢幕觀看。

## 積木面板詳解

積木面板位於畫面**左側**，提供四種積木類型，每種標題旁有 **ⓘ** 圖示，hover 可查看說明。

### 📦 模型變數

整個模擬都可存取的數值。每個變數有：

| 欄位 | 說明 |
|------|------|
| 名稱 | 英文識別符，例如 `x`、`vx`、`mass` |
| 型別 | 實數（小數）/ 整數 / 布林值（真/假）/ 文字 |
| 初始值 | 模擬開始時的值 |
| 說明 | 自訂備註（選填） |

### 🔵 微分方程

描述變數**隨時間的變化率**。格式：

```
d[變數名]/dt = 表達式
```

| 設定 | 說明 |
|------|------|
| 時間步長 dt | 每次計算前進的時間量，建議 `0.001`～`0.05` |
| 積分方法 | 見下表 |

**積分方法比較：**

| 方法 | 特點 | 適用場合 |
|------|------|----------|
| 歐拉法（簡易） | 誤差較大 | 學習用、快速測試 |
| **RK4 四階法（推薦）** | 精確，最常用 | 大多數物理模擬 |
| Verlet 積分（力學） | 能量守恆佳 | 彈簧、行星運動 |
| 自適應步長法 | 自動調整精度 | 剛性方程 |

### 🔴 計算/約束

每個時間步都重新執行的程式碼，用途包括：

- **計算衍生量**：`E = 0.5 * m * vx * vx + 0.5 * k * x * x;`
- **施加邊界條件**：`if (y < -4) { y = -4; vy = -vy; }`
- **預先計算受力**：`F = -k * x - b * vx;`（供 ODE 使用）

### 🟢 初始化

模擬**開始**（或按**重置**）時只執行一次的程式碼：

```js
x = 2.0;
vx = 0.0;
theta = Math.PI / 4;
```

與直接填「初始值」的差異：初始化可以寫計算式，例如從振幅和相位計算初始位置速度。

### 𝑓(𝑥) 數學函數速查

點擊面板底部的「**𝑓(𝑥) 數學函數速查**」展開速查表，點擊任何函數可複製語法到剪貼簿。

## 元件列表與屬性

在右側**元件列表**點擊「新增」可加入以下元件。點選元件卡片後，中央面板會開啟屬性編輯分頁。每個屬性旁有 **ⓘ** 說明，輸入欄旁的 **𝑓𝑥** 按鈕可從選擇器插入數學函數到游標位置。

右鍵點擊元件卡片可開啟快捷選單（查看屬性說明 / 開啟編輯 / 刪除）。

### Elements.DrawingPanel（繪圖面板）

所有 2D 元件的容器，**必須先新增此元件**才能看到其他元件。

| 屬性 | 說明 |
|------|------|
| 寬度 / 高度 | 畫布像素大小 |
| X 軸最小/最大值 | 畫布左右端對應的世界座標 |
| Y 軸最小/最大值 | 注意 Y 軸**向上為正** |
| 背景色 | 用英文如 `"white"`，或 `"#rrggbb"` 格式 |
| 鎖定等比例 | 開啟後圓形不會被拉成橢圓 |

### Elements.Shape2D（質點/形狀）

最常用的基本形狀，可用作質點、球、方塊等。

| 屬性 | 說明 |
|------|------|
| X / Y 位置 | 填入**變數名稱**（如 `x`、`y`）讓它動起來 |
| X / Y 半徑 | 單位為**世界座標**（非像素） |
| 形狀類型 | `ELLIPSE`（橢圓）/ `RECTANGLE`（矩形）/ `WHEEL`（輪子）/ `ROUND_RECTANGLE` |
| 旋轉角度 | 單位為**弧度**，填入變數可讓形狀旋轉 |
| 是否顯示 | 填入布林變數可動態控制顯示/隱藏 |

### Elements.Trail2D（軌跡線）

記錄質點運動路徑，自動在每幀記錄一個點連成線。

| 屬性 | 說明 |
|------|------|
| X / Y 位置 | 填入位置變數 |
| 最大點數 | 建議 500～2000，超過後自動刪除最舊的點 |

### Elements.Arrow2D（向量箭頭）

顯示向量，常用於力、速度、加速度的視覺化。

| 屬性 | 說明 |
|------|------|
| X / Y 起點 | 箭頭起點座標，通常填質點位置 |
| 向量X / 向量Y | 箭頭分量，填入速度或力的變數 |

### Elements.Spring2D（彈簧）

繪製螺旋彈簧圖形，常與 Shape2D 搭配使用。

| 屬性 | 說明 |
|------|------|
| X / Y 起點 | 彈簧固定端座標 |
| 端點X / 端點Y | 彈簧另一端座標，填質點位置讓彈簧伸縮 |

### Elements.Slider（滑桿）

讓使用者在模擬執行中拖動調整參數值。

### Elements.Button（按鈕）

點擊時執行指定的 JavaScript 程式碼（OnClick 欄位）。

### Elements.CheckBox（核取方框）

切換布林變數的狀態。

### Elements.Label（標籤）

顯示文字，`Text` 屬性可寫表達式，如 `"t = " + t.toFixed(2)`。

### Elements.ParsedField（數值輸入）

顯示並可編輯數值。`Format` 使用格式字串，如 `"0.000"` 表示顯示三位小數。

在任何數值欄位或 ODE 表達式中，可使用完整的 JavaScript `Math` 物件。也可點擊輸入欄旁的 **𝑓𝑥** 按鈕選擇並插入函數。

### 三角函數（角度均為弧度）

| 函數 | 說明 |
|------|------|
| `Math.sin(x)` | 正弦 |
| `Math.cos(x)` | 餘弦 |
| `Math.tan(x)` | 正切 |
| `Math.asin(x)` | 反正弦，回傳弧度 |
| `Math.acos(x)` | 反餘弦，回傳弧度 |
| `Math.atan(x)` | 反正切，回傳弧度 |
| `Math.atan2(y, x)` | 四象限反正切，回傳 -π 到 π |
| `Math.sinh(x)` | 雙曲正弦 |
| `Math.cosh(x)` | 雙曲餘弦 |
| `Math.tanh(x)` | 雙曲正切 |

> 角度轉弧度：`degrees * Math.PI / 180`

### 指數與對數

### 取值與比較

| 函數 | 說明 |

| `Math.E` | e ≈ 2.71828 |

| 按鈕 | 說明 |

## 檔案管理

### 儲存 .ejss 檔

點擊工具列「**儲存 .ejss**」，下載 XML 格式的模擬定義檔，可在之後重新載入繼續編輯。

## 注意事項與已知限制

### 座標系統

- 繪圖面板使用**數學座標系**：X 軸向右為正，**Y 軸向上為正**（與螢幕座標相反）。
- 所有元件的位置、大小屬性均以**世界座標**為單位，而非像素。

### 時間步長 dt 的選擇

| 情況 | 建議 dt |
|------|---------|
| 一般物理模擬（單擺、彈簧） | `0.01`～`0.02` |
| 高精度需求（三體、行星軌道） | `0.001`～`0.005` |
| 快速預覽（不要求精度） | `0.05` |

- **dt 只影響積分精度，不影響動畫播放速度**：系統會自動每幀執行多步以維持即時速度。
- dt 過大會導致積分不穩定，動畫出現抖動或數值爆炸。

### 角度單位

所有三角函數與旋轉屬性均使用**弧度（radians）**，非角度（degrees）。

```
45° = Math.PI / 4
90° = Math.PI / 2
180° = Math.PI
360° = 2 * Math.PI
```

### 變數命名規則

- 只能使用**英文字母、數字、底線**，且不能以數字開頭。
- 避免使用 JavaScript 保留字（`for`、`while`、`if`、`var`、`let`、`const` 等）。
- 內建時間變數 `t` 不需宣告，可直接在方程式中使用。

### DrawingPanel 是必要元件

所有 2D 元件（Shape2D、Trail2D、Arrow2D 等）**必須有 DrawingPanel 作為容器**才能顯示。新增任何 2D 元件前，請先新增一個 DrawingPanel。

### 計算/約束與 ODE 的執行順序

每個時間步的執行順序：

```
計算/約束 → ODE 積分（RK4 內部多步）→ 計算/約束（再次）→ 渲染
```

在約束中計算的中間變數（如合力 `F`）**在 RK4 的各個子步中不會重算**，對精度有輕微影響。若需最高精度，建議將力的公式**直接嵌入 ODE 的表達式**中，例如：

```
d[vx]/dt = (-k * x - b * vx) / m    ← 建議
```

而非：

```
// 約束中：F = -k*x - b*vx
d[vx]/dt = F / m                    ← 精度略低
```

### 三體等混沌系統

三體問題對初始條件極度敏感。即使使用 RK4 配合 `dt=0.001`，長時間後軌道仍會逐漸偏離理論解，這是**數值積分的固有誤差**，並非程式錯誤。縮小 dt（如 `0.0005`）可延長穩定時間。

### 匯出 HTML 的互動元件限制

匯出的 HTML 使用純 JavaScript 實作模擬核心，目前 `Slider`、`Button`、`CheckBox` 等互動元件在匯出版本中**不具備互動功能**，僅供顯示。

```
├── store/
│   └── simulationStore.ts   # Zustand 狀態
├── types/
│   └── simulation.ts        # TypeScript 型別定義
└── utils/
    ├── simulationRunner.ts  # HTML 產生器（預覽版 + 匯出版）
    └── ejssParser.ts        # .ejss 檔案解析器
```
